# Stage 2 -- Basic Forms

Time: 7 minutes

These are the fundamental moves of working with an AI coding tool. You should be able to do all of them without thinking. Today, practice them deliberately.

## First Move -- Ask Your Tool How It Works

Before anything else, find your tool's built-in guide. Every good tool has one. This is the most important skill -- learning to learn your tool from within.

| Tool Type | How to Find the Guide |
|-----------|----------------------|
| Claude Code | Type a question about how something works -- it will invoke a guide subagent |
| Gemini CLI | Ask "how do I do X in this tool" |
| Copilot CLI | `gh copilot --help` or ask it |
| Cursor | Open the command palette, look for Cursor-specific commands |
| VS Code Copilot | Open the chat panel, ask "what can you do" |
| Desktop apps | Just ask "what are your capabilities" or "how do I give you context" |

Spend 1-2 minutes here. Ask your tool at least one question about itself. Try things like:
- "How do I reference a specific file?"
- "How do I clear my context?"
- "What model are you using?"

## The Checklist

Work through as many of these as you can. You do not need to finish all of them. The goal is to practice, not to race.

### Getting Oriented

- [ ] Open your tool and start a conversation
- [ ] Ask your tool how it works (see above)
- [ ] Find out what model you are using
- [ ] Find out how to switch models if your tool supports it

### Context Control

- [ ] Reference a specific file in your conversation
- [ ] Find out how much context you have used (if your tool shows this)
- [ ] Clear your context and start fresh
- [ ] Find out how to add a folder or project as context

### Session Management

- [ ] End your current session
- [ ] Start a new session
- [ ] Resume a previous session (if your tool supports it)
- [ ] Find your conversation history

### Steering

- [ ] Stop something in progress (cancel a generation)
- [ ] Ask the tool to undo or revert its last change
- [ ] Ask the tool to explain what it just did
- [ ] Give the tool a correction ("no, I meant X not Y")

### Awareness

- [ ] Find out how much usage you have left (quota, credits, rate limits)
- [ ] Find out what permissions your tool has (can it edit files? run commands?)
- [ ] Find out what your tool cannot see (files outside workspace, private repos, etc.)

## What to Notice

- Which of these were easy and which were hard?
- Did any of them not work in your tool?
- Which ones will change how you work day-to-day?
- What question did you ask your tool about itself, and was the answer useful?

## Going Deeper (Solo Practice)

If you are doing this solo and want to spend more time here, try these:

- Practice every item on the checklist until you can do it in under 3 seconds
- Try the same checklist in a different tool and compare
- Find 3 capabilities of your tool that are not on this checklist
- Ask your tool "what is the thing most people do not know about you"
